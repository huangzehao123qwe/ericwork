The Problem

Confusion & lengthy support turnaround

Procurement teams were drowning in a backlog of routine "where do I find X?" queries. Support turnaround was slow, and employees had no confident path forward for purchase decisions — leading to delays, escalations, and frustrated end-users.

The Vision

No wait. 100% accurate suggestions.

An intelligent intake agent that understands requester intent, suggests the right route, and gives procurement managers the admin tools to configure, test, and optimize — all with zero hallucination risk.

Scope · Today's Focus

Requester experience & admin control surface

Requester — LLM Intent Understanding Natural language processing to understand what the requester actually needs.
Requester — Route Suggestion LLM + route logic to surface the best path for each purchase request.
Admin — Route Table Configuration Intuitive admin console for configuring routing rules and fallback responses.
Admin — LLM Query Playground A testing sandbox so admins can validate LLM responses before they go live.
Admin — Data Dashboard Analytics to monitor route performance and continuously optimize decisions.
Future — Predictive Requests Predict upcoming requests based on user group patterns (next phase).
Design Evolution

From ChatGPT-like to a hybrid experience

Early prototypes mimicked a general chat interface — users had to know what to ask. We shifted to a hybrid model that meets both casual and power users where they are.

Before

ChatGPT-like Experience

  • ML-generated sample questions only
  • Users must know what to ask
  • No admin-defined guidance
  • One-size-fits-all for all roles
After

Hybrid Experience

  • ✓ Admin-defined Quick Links for common actions
  • ✓ ML-generated Q&A tailored by user role
  • ✓ Designed for both casual & power users
  • ✓ Trust-first — accuracy is non-negotiable
Core Design Principle

Trust building is key.

Financial and procurement decisions require absolute accuracy. We designed explicit transparency mechanisms — source citations, confidence indicators, and admin-controlled fallback responses — to ensure users never had to guess whether to trust the agent.

Research & Insights

What we learned from procurement teams

  • 📊
    Procurement admins spend 50%+ of their time answering basic queries like "where do I find X?" — instead of focusing on strategic work.
  • 📚
    Internal documentation and training materials become outdated quickly, requiring costly, time-consuming retraining cycles.
  • 🔒
    Financial support demands accuracy above all else. Procurement teams distrust bots and LLMs due to hallucination risk — making transparency and accuracy non-negotiable design constraints.

Key Highlights

What made this project stand out

01

The hybrid model beat pure chat by meeting every user where they are

Blending admin-defined Quick Links with ML-generated Q&A served both first-time requesters and power users without sacrificing either experience.

02

Admin tooling is as critical as the end-user experience

Without a robust configuration console, query playground, and fallback system, trust in the agent would have broken down before launch.

03

Trust in AI isn't promised — it's earned through explainability at every step

Source citations, confidence signals, and admin-controlled fallbacks were the design decisions that turned skeptics into advocates.

Reflection

"Context isn't just another table of data — it's part of the experience of creating a compliant procurement group. Aligning rules with users is such a game changer."

Next steps: After collecting initial early-adopter feedback, the roadmap includes Live Preview (so admins can see changes before publishing) and ML feedback loops to intelligently suggest rules based on user behavior patterns.
