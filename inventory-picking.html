Context & Problem

Warehouse workers spending more time clicking than picking

A warehouse stock-picking workflow on Workday's mobile app had become a bottleneck for high-volume operations. Workers were fighting the interface instead of focusing on the work itself — causing delays, errors, and frustration on the floor.

Root Causes

Three compounding issues were identified: missing contextual information left workers guessing; outdated design system components built for low-density workflows created excessive whitespace and scrolling; and a click-heavy workflow model that required constant tapping just to progress.

Usability Audit

Why the old design failed

  • 📜
    Excessive information density issues — a huge banner consumed prime screen real estate, making it hard to know where to start or what had been picked.
  • 📦
    Components built for low-volume workflows — large padding, wide input fields, and excessive whitespace made the interface unfit for fast-paced, high-volume warehouse tasks.
  • 🔎
    Unclear hierarchy — users couldn't immediately identify the primary action (picking) from secondary information, creating cognitive friction on every interaction.
  • 👆
    Scroll, click, scroll, click — key actions were buried under visual noise, forcing a repetitive, error-prone pattern for every single pick.
Issue 1 — System-Level Inefficiency

Components built for the wrong density

The design system components were engineered for low-density, general-purpose workflows — not the high-speed, bulk-operation context of warehouse picking. This created excessive scrolling, unclear hierarchy, and low efficiency precisely where speed matters most.

Issue 2 — Missing Key Actions

Task actions buried under visual noise

Critical task actions — like confirming a pick, skipping an item, or marking a bin complete — were buried deep in the visual hierarchy. The screen was overloaded with secondary information, making primary actions hard to find under time pressure.

Design Iterations

From categorized lists to scanner-driven UX

Iteration 1

High-Density List + Categorized Items

Replaced the legacy task view with a high-density list, removed the oversized banner, and grouped items by aisle and bin to reduce scanning time. Problem: Same scroll-and-click behavior, key actions still unclear.

Iteration 2 → Final

Single-Item Scanner-Driven Flow

Showed one item at a time — "Now Picking / Up Next" — driven by bin scanning. Added clear user to-do actions, automated quantity pre-fill, and a numeric keypad confirmation step. Result: Massively reduced clicks, maximum focus on the current task.

Final Solution

A balanced, scalable mobile-first task flow

  • Less clicks, more scans — scanner-driven progression eliminated most touch interactions
  • Less data, more context — surface only what's needed at each step
  • Automated quantity pre-fill — reduced manual input by predicting expected quantities
  • Led Design System component updates for the Financial use case as part of this work
Key Highlights

What made this project stand out

01

Designing for high-speed environments requires rethinking density, not just layout

The problem wasn't the visual design — it was components built for the wrong use case. Fixing density unlocked everything else.

02

Scanner-driven thinking unlocked a completely different UX paradigm

Shifting from "show all items, let the user navigate" to "show one item, let the scanner drive" cut cognitive load and interaction steps simultaneously.

03

Constraints on a small screen force better decisions about what actually matters

Every element had to justify its presence. The discipline of mobile IA made the design sharper — and the principles transferred directly to the Design System.

Reflection

"Simplifying doesn't mean reducing — it means focusing on what drives action."

Designing information architecture within a tight mobile screen pushed me to think deeply about info hierarchy and clarity. Every pixel had to earn its place. The biggest insight: constraints aren't limitations — they're the forcing function for better decisions.
